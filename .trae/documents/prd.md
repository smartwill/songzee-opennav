# 产品需求文档 (PRD) — 开源软件替代品目录 (OpenAlternative 克隆)

## 1. 产品概述
一个开源软件替代品目录网站，高度还原 openalternative.co 的形态与体验。
- 解决问题：帮助用户发现商业/闭源软件的开源替代方案。
- 目标用户：开发者、技术爱好者、注重隐私与成本的用户。
- 市场价值：聚合「商业软件 → 开源替代」的映射，降低开源选型门槛。

## 2. 核心功能

### 2.1 功能模块
1. **首页**：Hero 区 + 按字母排序的软件目录 + 分类导航 + 统计数据。
2. **替代品详情页** (`/{slug}`)：展示某商业软件的开源替代列表（卡片、徽章、GitHub 数据）。
3. **分类页** (`/category/{slug}`)：按分类浏览所有商业软件。
4. **搜索页** (`/search?q=`)：实时检索软件、工具、分类。
5. **工具详情页** (`/tool/{slug}`)：单个开源项目详情与「可替代的软件」。
6. **关于页** (`/about`)：项目说明。

### 2.2 页面详情
| 页面 | 模块 | 功能描述 |
|------|------|----------|
| 首页 | Hero 区 | 标题、副标题、统计数字、主搜索框、CTA |
| 首页 | 软件目录 | 字母索引 + 网格/列表，显示名称与替代数量 |
| 首页 | 分类导航 | 分类标签云，按分类筛选 |
| 替代品详情页 | 头部 | 商业软件名称、Logo、简介、官网链接 |
| 替代品详情页 | 替代列表 | 开源工具卡片：名称、描述、Star、License、语言、官网/仓库 |
| 替代品详情页 | 相关分类 | 所属分类与相关软件 |
| 分类页 | 列表 | 该分类下所有商业软件 |
| 搜索页 | 结果 | 软件与工具混合结果，高亮关键词 |

## 3. 核心流程
用户进入首页 → 通过搜索或字母索引/分类找到目标商业软件 → 进入替代品详情页 → 查看开源替代卡片 → 点击外链访问官网/仓库。

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：温暖橙红 (#f97316 系) 作为强调色，中性暖灰为基底；深色模式以深石板灰为底。
- **辅助色**：开源生态绿点缀（成功/Star 徽章）。
- **字体**：标题用 `Sora`（几何感展示字体），正文用 `Plus Jakarta Sans`（圆润易读），代码/数字用 `JetBrains Mono`。
- **布局**：顶部粘性导航 + 卡片网格，圆角 14px，柔和投影，卡片悬浮抬升。
- **图标**：使用内联 SVG 图标（Star、Git、External、Search、Sun/Moon）。
- **氛围**：浅色背景叠加微妙网格/噪点纹理；Hero 区柔和渐变光晕。

### 4.2 页面设计概览
| 页面 | 模块 | UI 元素 |
|------|------|---------|
| 首页 | Hero | 大标题渐变文字、统计胶囊、搜索框、背景光晕 |
| 首页 | 目录 | 字母锚点 + 软件芯片网格，悬浮卡片 |
| 详情页 | 列表 | 工具卡片，左侧 Logo，右侧徽章组，悬浮高亮 |

### 4.3 响应式
桌面优先，移动端自适应（网格 1→2→3 列），导航折叠为汉堡菜单，触摸友好的点击区域。
